# Bottom Navigation Flow

## Goal

Provide simple, predictable navigation between primary modules.

## Main Tabs

- Home
- Customers
- Cashbook
- Reports

## Rules

- Active tab must be visually clear
- Navigation should feel stable and consistent
- Each tab should preserve context where possible

## Tab Purpose

- Home -> overview and action hub
- Customers -> manage customer list
- Cashbook -> money in/out entries
- Reports -> insights and collection-focused reporting

## Notes

- Bottom navigation must remain clean and readable
- Avoid too many tabs
