# Global Product Rules

## Product Identity

Baqaya is an offline-first ledger app designed to help small businesses track balances and collect payments faster.

## Core UX Principles

- Mobile-first
- Offline-first
- Fast actions
- Simple UI
- Clear empty states
- Strong reusable patterns
- Minimal friction
- Action-oriented screens

## Design Rules

- Keep screens readable
- Prioritize important actions
- Reuse components wherever possible
- Use consistent spacing and hierarchy
- Avoid unnecessary clutter

## Engineering Rules

- Build reusable components first
- Avoid screen-specific duplication
- Keep route structure clear
- Match design but prefer usability when needed

## Navigation Rules

- Auth separate from app tabs
- Onboarding after first login
- Main usage centered around Home, Customers, Cashbook, Reports

## Reminder Rules

- Reminders open WhatsApp with customer number
- Never require owner WhatsApp number in auth

## Offline Rules

- Save locally first
- Sync later
- Show lightweight sync awareness
- Do not make syncing the user’s main task

## Goal of the App

The app should feel like a simple tool that helps the user take action quickly, not a heavy system they have to manage.
