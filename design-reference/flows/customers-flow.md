# Customers Flow

## Goal

Let users view, search, manage, and act on customers easily.

## Main Flow

1. User opens Customers tab
2. App shows customers list
3. User can:
   - View customer
   - Add customer
   - Import customer
   - Send reminder
   - Call customer

## States

- Empty state
- Customer list with data
- Offline state
- Pending sync indicator if relevant

## Empty State

- Show message encouraging user to add first customer
- Primary action = Add Customer
- Secondary action = Import from Contacts

## List Item Actions

- Open customer details
- Quick reminder action
- Optional call/WhatsApp

## Notes

- Customer screen should feel fast and useful
- Keep list clean and readable
