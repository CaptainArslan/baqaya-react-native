# Reports Flow

## Goal

Help user understand what money to collect and who needs attention.

## Main Flow

1. User opens Reports tab
2. User sees:
   - total baqaya
   - late payers
   - top debtors
   - period tabs (today / week / month)
   - summary stats

## Period Tabs

- Today
- Week
- Month

## Priority Order

1. Amount to collect
2. Late payers
3. Top debtors
4. Stats and trends

## States

- Empty reports
- Reports with data
- Partial data
- Offline local data

## Rules

- Reports should be action-driven
- Focus on collection, not just display
- Hide meaningless charts if no data

## Notes

- The user should know who to follow up with next
