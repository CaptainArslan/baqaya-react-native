# Customer Details Flow

## Goal

Show a single customer’s balance, history, and available actions.

## Main Flow

1. User opens customer details
2. Show:
   - name
   - phone
   - current balance
   - recent transactions
   - action buttons

## Actions

- Add Entry
- Send WhatsApp Reminder
- Call Customer
- Edit Customer

## States

- No transactions yet
- Has transactions
- Offline data available
- Sync pending or failed if relevant

## Notes

- This screen should feel like an action hub
- Balance and reminder action should be prominent
